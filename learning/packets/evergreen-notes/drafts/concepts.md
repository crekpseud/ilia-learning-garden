# Candidate Concepts

## Evergreen Note

- Status: promoted to [[wiki/concepts/evergreen-note]]
- Definition: A note written and organized to evolve, contribute, and accumulate insight over time, serving as a reusable building block for thought.
- Why it matters: It may define the note-level practice behind trusted concept notes: atomic, concept-oriented, densely linked, and written in the user's own words.
- Possible aliases: living note, atomic concept, Zettel, evergreen note-writing
- Related existing notes: [[wiki/concepts/digital-garden]], [[wiki/synthesis/digital-garden-ethos]], [[wiki/synthesis/technopastoral-synthesis]], [[docs/adr/0016-use-topographical-reading-with-append-only-process-history]]
- Promotion recommendation: yes, pending human-contact review
- If existing, suggested patch: N/A

## Bi-directional Linking

- Status: deferred
- Definition: A linking mechanism that makes both destination and source visible, creating two-way contextual navigation.
- Why it matters: It supports topographical navigation and reveals relationships that were not explicitly planned.
- Possible aliases: backlinks, two-way links, inbound links
- Related existing notes: [[wiki/synthesis/technopastoral-synthesis]], [[wiki/concepts/digital-garden]]
- Promotion recommendation: maybe
- If existing, suggested patch: N/A

## Epistemic Status

- Status: deferred
- Definition: Metadata indicating the level of maturity, certainty, or done-ness of a note.
- Why it matters: It may help distinguish rough ideas from trusted insights, but it could also add ritual overhead.
- Possible aliases: maturity level, note status, growth stage
- Related existing notes: [[wiki/synthesis/llm-wiki-pattern]], [[wiki/concepts/digital-garden]]
- Promotion recommendation: maybe
- If existing, suggested patch: N/A

## The Ritualization Trap

- Status: promoted to [[wiki/concepts/the-ritualization-trap]]
- Definition: The failure mode where knowledge practices become self-optimizing maintenance rituals that distract from reading, thinking, and learning.
- Why it matters: It names the main tension in adopting evergreen notes inside a playful garden protocol.
- Possible aliases: polishing trap, optimization trap, maintenance trap
- Related existing notes: [[wiki/concepts/digital-garden]], [[wiki/synthesis/digital-garden-ethos]]
- Promotion recommendation: not yet
- If existing, suggested patch: N/A

## Note Titles as APIs

- Status: promoted to [[wiki/concepts/note-titles-as-apis]]
- Definition: The idea that note titles act as declarative handles or interfaces for retrieving, linking, and composing thoughts.
- Why it matters: It names a practical bridge between human navigation and AI-readable Markdown: a good title becomes an API for both memory and tooling.
- Possible aliases: title handle, thought handle, note API
- Related existing notes: [[wiki/synthesis/llm-wiki-pattern]], [[wiki/concepts/digital-garden]]
