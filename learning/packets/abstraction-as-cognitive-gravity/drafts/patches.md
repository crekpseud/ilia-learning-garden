# Suggested Patches

## Patch: [[wiki/synthesis/object-thinking-and-the-digital-garden]]

Rationale:
This packet suggests that objects, notes, and skills may all act as cognitive attractors: stable centers that gather behavior, meaning, and future work.

Suggested addition:

```markdown
Object Thinking also fits a broader abstraction pattern: a good abstraction acts like a cognitive attractor. It gathers scattered procedures, data, responsibilities, and examples into a stable conceptual basin. In that sense, an object is not merely a programming unit; it is a local center of gravity for behavior.
```

## Patch: [[wiki/concepts/object-thinking]]

Rationale:
The gravity metaphor could sharpen why Object Thinking emphasizes responsibility and behavior over data shape.

Suggested addition:

```markdown
An object can be read as a cognitive attractor: a named center where related responsibilities, behaviors, collaborators, and expectations gather. This is why Object Thinking resists treating objects as passive data containers.
```

## Patch: [[wiki/synthesis/original-thought-as-active-reconstruction]]

Rationale:
The source conversation demonstrates active reconstruction as repeated movement from formal rule to conceptual meaning.

Suggested addition:

```markdown
Active reconstruction often starts with small formal puzzles. Asking what a conversion, expression, operator, or runtime lowering means can become a route into deeper architectural understanding when the learner rebuilds the abstraction behind the rule.
```

## Patch: [[wiki/concepts/knowledge-gardener]]

Rationale:
The packet introduces `Map Of Curiosity` as a gardener artifact for preserving trails before promoting concepts.

Suggested addition:

```markdown
A knowledge gardener may create maps of curiosity: lightweight, untrusted artifacts that preserve question trails and possible growth directions before deciding which notes deserve promotion.
```

## Protocol Design Questions

- Should `Map Of Curiosity` become a trusted protocol concept after several real uses?
- Should AI conversation captures have a normalized packet operation distinct from NotebookLM packet imports?
- Should long shared chats preserve a transcript excerpt, a source record, or only a map and packet export?
- How much of a private ChatGPT conversation can safely be seed-promoted as a synthetic example?
