# Suggested Patches

## Patch: [[wiki/synthesis/llm-wiki-pattern]]

Rationale:
This packet adds a caution to the LLM wiki pattern: AI-generated or AI-maintained wiki material should support human reconstruction, not replace it.

Suggested addition:

```markdown
An LLM-maintained wiki should not optimize only for frictionless synthesis. If the system removes all cognitive effort, it can produce an illusion of competence: the learner can navigate polished notes without being able to retrieve, explain, or apply the underlying ideas. The wiki should therefore preserve human-contact artifacts that show active reconstruction.
```

## Patch: [[wiki/synthesis/digital-garden-ethos]]

Rationale:
This packet links garden tending to active reconstruction and desirable difficulty.

Suggested addition:

```markdown
Tending a digital garden should include active reconstruction, not only curation. Practices such as retrieval, teach-back, first-principles questioning, and spaced return make the garden slower to grow in the short term, but help prevent familiarity from masquerading as understanding.
```

## Patch: [[wiki/concepts/digital-garden]]

Rationale:
This packet clarifies a risk for digital gardens that are heavily augmented by AI assistants.

Suggested addition:

```markdown
An AI-augmented digital garden should distinguish scaffolding from outsourcing. AI can help gather, compare, and draft, but trusted garden knowledge should still pass through the gardener's own reconstruction before promotion.
```
