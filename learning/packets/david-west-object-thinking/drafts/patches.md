# Suggested Patches

## Patch: [[wiki/synthesis/digital-garden-ethos]]

Rationale:
Object Thinking gives the garden's living-system metaphor a software-design counterpart: software as a community of collaborating objects rather than a machine built from passive parts.

Suggested addition:

```markdown
David West's Object Thinking offers a software-design parallel to the digital garden metaphor. It treats software less as a machine assembled from passive parts and more as a community of collaborating objects with responsibilities, behaviors, and contextual roles. The metaphor is useful when it improves domain understanding, but it must still survive contact with implementation practice.
```

## Patch: [[wiki/concepts/digital-garden]]

Rationale:
The packet links the digital garden's networked, contextual structure to West's view of a system as patterned interaction between participants.

Suggested addition:

```markdown
The garden can also be read as a community of interacting objects: notes, links, packets, sources, and agents each carry responsibilities and participate in larger patterns. This analogy is useful only when it clarifies behavior and relationships rather than turning software design into decorative metaphor.
```
