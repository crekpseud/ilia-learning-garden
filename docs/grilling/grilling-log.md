## 2026-06-21: Protocol And MVP Shape

Resolved:

- The MVP proves the daily learning loop through a concrete vault population loop.
- A learning packet is the first-class input unit.
- Learning packets live inside the vault, outside the trusted wiki.
- Human-contact artifacts are required before promotion.
- Promotion creates one synthesis note by default and selected reusable concept notes only when approved.
- Existing trusted notes can receive suggested patches only.
- The first NotebookLM bridge uses Google Docs packet exports, imported manually by URL before any Drive folder automation.
- The system is a portable Markdown-first protocol. Obsidian is the first IDE, Codex is the first automation client, NotebookLM is a replaceable synthesis server, and Google Drive is a replaceable bridge.
- The repository is the seed vault and uses protocol-first top-level folders.
- The vault is Markdown-first with attachments allowed in raw or learning areas, but not in the trusted wiki.
- The first milestone is a protocol-only proof before building the Google Docs bridge.
- Trusted knowledge should be read topographically, while workflow history remains append-only and trusted note changes happen through reviewed patches.
