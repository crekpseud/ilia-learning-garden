# Use this repository as the seed vault

This repository will start as the first living vault and protocol template rather than only tooling around a separate future vault. Project documentation, schema, learning packets, trusted wiki notes, and eventual tools can co-evolve here; tooling can be extracted later if reuse demands it.

**Considered Options**

- Treat this repository as the vault.
- Treat this repository as tooling that operates on a separate vault.
- Start as a seed vault and extract tooling later.
- Make the repository a reusable template immediately.

**Consequences**

Repository structure should be designed as an Obsidian-compatible Markdown codebase first. Any future CLI, app, or integration should operate on that structure rather than replacing it.
