# Note Titles as APIs

## Definition

Note Titles as APIs is the practice of naming notes so their titles become stable handles that humans, links, and agents can use to retrieve, compose, and extend thoughts.

## Why It Matters

In a Markdown garden, a title is more than a label. It is an interface. A good title makes an idea invokable: it can be linked, remembered, searched, passed into prompts, and used as context by an agent.

This gives [[wiki/synthesis/llm-wiki-pattern]] a human-facing naming principle. If Markdown notes are durable context for future AI-assisted work, titles become part of the API surface of the garden.

The concept also sharpens [[wiki/concepts/evergreen-note]] practice. An evergreen note needs a title that can survive reuse. But title design can also become another version of [[wiki/concepts/the-ritualization-trap]] if naming gets optimized before the idea has earned a stable handle.
