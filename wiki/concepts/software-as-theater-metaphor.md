# Software-as-Theater Metaphor

## Definition

Software-as-Theater Metaphor is a software-design metaphor where objects are actors with intrinsic talents, applications provide roles and scripts, and the environment acts as a stage.

## Why It Matters

The metaphor helps separate what an object is capable of from the role it plays in a particular application situation. That distinction matters because a domain object should not be permanently typecast by one workflow, screen, transaction, or procedural use case.

Inside [[wiki/concepts/object-thinking]], the theater metaphor supports a richer view of collaboration. Objects are not inert records; they participate in scenes, respond to messages, and take on contextual roles.

The metaphor should remain practical. It is useful when it clarifies responsibility and interaction. It becomes a problem when it replaces implementation judgment with theatrical language.
