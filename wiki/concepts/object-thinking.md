# Object Thinking

## Definition

Object Thinking is a philosophical and behavioral approach to software development that models software as a community of autonomous, collaborating objects rather than passive data manipulated by procedures.

## Why It Matters

Object Thinking shifts attention from classes, inheritance, and syntax toward responsibility, behavior, message passing, and domain meaning. It asks what the system is made of in human and domain terms before asking how the code should be arranged.

For this garden, Object Thinking matters as the first trusted software-design concept. It links the garden's living-system metaphors to software practice, but it should stay a modeling lens rather than a doctrine. The user's reflection explicitly keeps object, function, text, actor, and agent metaphors in play.

Object Thinking is useful when it improves responsibility assignment, domain modeling, and collaboration between parts. It becomes suspect when it turns into beautiful language that cannot survive implementation practice.
